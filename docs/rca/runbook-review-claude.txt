  5. ADEQUACY: Overall Assessment

  What these tests catch well:

  - DOM element existence and basic structure
  - API endpoint responses
  - Settings persistence
  - Tab management (open/close/switch)
  - WebSocket connectivity and reconnection
  - Claude CLI slash commands
  - Session lifecycle (create/archive/hide/rename)

  Bug categories that would slip through:

  1. Gemini/Codex functional bugs: Any bug in the Gemini CLI wrapper, Codex CLI wrapper, or their terminal I/O is completely undetectable. The tests create these
  sessions but never use them.
  2. Conversation persistence bugs: If conversation history is lost on tab close/reopen or page refresh, no test catches it reliably. CLI-19 has a weak check.
  3. Real user interaction bugs: Many tests use dispatchEvent and .value= instead of actual clicking and typing. Bugs triggered by real user interaction (focus
  management, event bubbling, click handlers not attached) would pass these tests but fail for users.
  4. File editor workflow bugs: EDIT-01 through EDIT-07 in Phase 12 are one-line descriptions with no actual test steps. The file editor (CodeMirror/Toast UI) has
  virtually no functional testing — saving, editing, dirty tracking, Save As, close-with-unsaved-changes are all just placeholders.
  5. Cross-browser/responsive bugs: No viewport size testing. No mobile testing.
  6. Race conditions: Tests use fixed browser_wait delays (2-5s). Real race conditions between WebSocket messages, state updates, and DOM rendering aren't caught by
  sleeping.
  7. Error handling UI: No test triggers a server error (500) and verifies the UI shows a meaningful error message rather than failing silently.
  8. Concurrent session activity: No test has two sessions actively receiving responses simultaneously and checks for cross-contamination beyond CLI-21's basic
  isolation check.
  9. Voice input end-to-end: Only the button and WebSocket endpoint are verified. The actual recording-to-terminal flow is untested.
  10. Project config modal Save button: USR-07 notes "#cfg-save button click via JS didn't persist state — state change done via API directly." This suggests the
  project config save button may be broken, and the workaround masked the bug.

  Phase 12 quality concern

  Phase 12 (lines 2726-2857) contains 32 test blocks but most are one-liner descriptions without steps, expected results, or verification criteria. They look like a
  checklist of features to test, not actual tests. Tests like EDIT-01 through EDIT-07, TASK-01 through TASK-06, CONN-01/02, and MCP-01 through MCP-07 are stubs.

  The runbook is thorough for Claude CLI interactions and basic UI structure (Phases 1-7 are solid). However, it has three critical gaps: (1) zero Gemini/Codex
  interactive testing, (2) Phase 12 tests are stubs not real tests, and (3) conversation persistence across close/reopen is not verified as a coherent workflow for any
  CLI type.
